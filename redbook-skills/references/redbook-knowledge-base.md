# RedBook Knowledge Base

目标：把分析和执行沉淀成可复用的 markdown。

## 目录

```text
redbook-knowledge-base/
  README.md
  accounts/
  topics/
  patterns/
  actions/
  reviews/
```

## 类型说明

- `accounts/`：账号定位、竞品分析、体检
- `topics/`：选题、标题骨架、争议点
- `patterns/`：爆款结构、封面模板、互动机制
- `actions/`：抓取、发布、回复、下载、复刻
- `reviews/`：复盘、失败原因、回退策略

## 每条记录最少包含

- 一句话结论
- 证据或来源
- 可复用点
- 风险或边界
- 下一步动作

## 写入时机

1. 任务前先检索
2. 任务中记临时发现
3. 任务后写正式结论
4. 失败时单独记阻塞和回退

## 文件命名

- 用 `YYYY-MM-DD-brief.md`
- brief 保留 2-6 个高信息量词

示例：

- `2026-03-27-title-hook-checklist.md`
- `2026-03-27-account-positioning-review.md`
