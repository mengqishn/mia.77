# RedBook Browser via Playwright

用本地浏览器而不是 OpenClaw 时，默认走 `Playwright + 本机 Chrome 登录态`。

## 目标

- 读取首页推荐流
- 打开指定笔记 URL
- 抽取标题、正文、作者、互动数据、图片链接
- 自动进入创作页、填写标题正文、上传素材
- 自动进入通知页、逐条回复评论

## 首选模式

优先使用持久化浏览器上下文：

- 复用本机 Chrome 用户目录
- 保留登录态、Cookie、本地存储
- 避免每次重新登录

## 运行顺序

1. 启动持久化上下文。
2. 打开目标 URL 或首页。
3. 校验页面状态：已登录、非 404、存在标题或互动区。
4. 优先 `page.evaluate()` 提取结构化字段。
5. 关键节点才截图。
6. 单步动作最多重试 1 次。

## 提取优先级

- 标题
- 作者
- 发布时间
- 正文内容
- 点赞 / 收藏 / 评论
- 图片链接
- 标签 / 话题

## 发布动作优先级

- 确认到创作后台
- 上传素材
- 填写标题
- 填写正文
- 校验封面与话题
- 默认停在“发布”按钮前

## 评论回复动作优先级

- 先到通知页
- 再对位评论内容
- 再生成回复
- 再输入发送

## 失败降级

- 页面结构变化：先更新 selector，再继续。
- 页面长时间空白：刷新一次，仍失败则停。
- 登录失效：提示用户人工登录后继续。
- 发布按钮不可点：保留已填写内容，停在当前页汇报。

## 不要做的事

- 不要在未验证登录态时盲跑流程。
- 不要编造页面上看不到的字段。
- 不要对同一个失败动作无意义重试。
- 不要直接逐字复刻原文和原图。
